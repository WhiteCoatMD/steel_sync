## 7. Common Frustrations with Existing Configurators

### Technical Frustrations
- **Slow loading**: If tool doesn't load in a few seconds, browser may not support it
- **Browser compatibility**: WebGL not supported on all older devices
- **Mobile performance**: Many configurators perform poorly on phones
- **Limited customization**: One-size-fits-all approaches frustrate users with individual needs

### Business Process Frustrations
- **No integrated ordering**: Configurator generates a design but ordering is separate
- **Pricing inaccuracy**: Quotes from configurator don't match final price
- **Limited backend control**: Dealers can't update pricing/options themselves
- **Poor dealer support**: Manufacturers don't respond to dealers after initial sale
- **Broker vs dealer confusion**: Online tools used by brokers who disappear after sale

### Sales Process Frustrations
- **High-pressure tactics**: Configurator used as high-pressure sales funnel
- **No follow-up automation**: Leads captured but no automated nurturing
- **Manual handoff**: Design created in configurator but must be re-entered into order system
- **Error-prone manual processes**: Hand-drawn sketches and manual calculations still common
- **Expensive tools**: MBS software has steep price tags; SmartBuild at $549/month

### Customer Experience Frustrations
- **Not enough visualization**: Static images don't show complete picture
- **Complex interfaces**: Too many options without guidance
- **No AR/on-site preview**: Can't visualize building in actual location
- **Delivery delays**: Companies wait for convenient installer routes

## 10. Technical Blog Posts & Documentation on Building 3D Configurators

### Architecture Overview

**Frontend Layer**
- WebGL-based frameworks: Three.js or Babylon.js for real-time rendering
- React Three Fiber: preferred for React teams (combines 3D + modern UI + state management)
- Model formats: GLTF/GLB (web-optimized, compressed binary)
- PBR (Physically Based Rendering) materials for realistic appearance

**Backend Layer**
- API layer for pricing logic, product data, rule validation
- Configuration rules engine ensuring every combination is valid, buildable, priced correctly
- CMS for managing assets, pricing, variants without code changes
- Cloud hosting + CDN for global performance

**Asset Pipeline**
- Import modular GLTF/GLB models (low-poly, optimized)
- Apply PBR materials, texture swapping, color customization
- Real-world material references (siding, shingles, trim, doors, windows)
- LOD (Level of Detail) techniques for performance
- GPU instancing for complex scenes
- Compressed textures

**Integration Layer**
- eCommerce platforms (Shopify, Magento, WooCommerce)
- ERP/PIM systems for inventory/pricing sync
- CRM or CPQ tools for quote generation
- Analytics systems (conversion tracking)
- AR via ARKit/ARCore

### Shed Configurator Specific Architecture (from Three.js Forum Job Post)
- **Input**: 2D floorplan layouts (SVG-like schematic)
- **Conversion**: 2D layout logic converted to interactive 3D
- **Components**: Reusable building elements
- **Rules**: Geometry placement rules (doors, windows, vents, ramps, loft areas)
- **Assets**: Modular GLTF/GLB with PBR materials
- **Tools**: Blender/3ds Max for modeling, optional GLSL shaders

### Development Estimates
- **Team needed**: 3D artists, frontend devs, backend engineers, UX designers, DevOps
- **MVP**: 2,000-3,000 hours ($350K-$525K)
- **Production-ready**: 3,000-6,000 hours ($350K-$1M+)
- **Ongoing**: CMS, asset updates, pricing changes, new features
